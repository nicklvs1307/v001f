feat(superadmin): Implement Login as Tenant feature

This commit introduces the "Login as Tenant" feature, allowing superadmins to impersonate tenant admins for debugging and support purposes.

Backend:
- Created a new endpoint `/api/superadmin/login-as-tenant/:tenantId` to generate a JWT for a tenant user.
- Added a new `superadminController.js` with the `loginAsTenant` function.
- Registered the new `superadminRoutes.js` in the main router.

Frontend:
- Added a "Login as" button to the tenants list in `TenantList.js`.
- Implemented the `handleLoginAsTenant` function to call the new endpoint, store the original token, and set the new tenant token.
- Created a `ReturnToSuperadminBanner.js` component that is displayed when impersonating a tenant.
- The banner includes a "Return to Superadmin" button that restores the original superadmin session.
- Added the `superadminService.js` to handle the API call.
